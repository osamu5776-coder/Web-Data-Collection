# Web Data Collection Project

## プロジェクト概要

Webからデータを収集・管理するプロジェクト。

## Git 運用ルール

**コードを変更するたびに、必ずGitHubへプッシュすること。**

### 基本フロー

1. コードを変更する
2. 変更内容を確認する: `git diff`
3. ステージングに追加する: `git add <ファイル名>`（`git add -A` は避ける）
4. コミットする: `git commit -m "変更内容の説明"`
5. **即座にプッシュする**: `git push`

### コミットメッセージ規則

- 変更の「なぜ」を中心に書く
- 日本語・英語どちらでも可
- 例:
  - `fix: スクレイピング時のタイムアウトエラーを修正`
  - `feat: 新しいデータソースのパーサーを追加`
  - `refactor: データ保存ロジックを整理`

### 注意事項

- `.env` や認証情報を含むファイルは絶対にコミットしない
- 大きなバイナリファイル（収集したデータファイルなど）は `.gitignore` で除外する
- 破壊的な操作（`--force` push、`reset --hard` など）は必ず確認してから実行する

## ディレクトリ構成（予定）

```
Web Data Collection/
├── CLAUDE.md
├── .gitignore
├── README.md
├── src/           # ソースコード
├── data/          # 収集データ（.gitignore 対象）
└── tests/         # テスト
```

## 開発方針

- セキュリティ: スクレイピング対象サイトの利用規約を遵守する
- エラーハンドリング: ネットワークエラーやレート制限に対応する
- ログ: データ収集の実行ログを記録する
